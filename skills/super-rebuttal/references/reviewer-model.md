# Reviewer Model

Use reviewer cards as an internal reasoning layer before drafting.

Each reviewer card should capture:

- `reviewer_id`
- `sentiment`: positive / mixed / negative
- `movability`: supportive / swing / fixed
- `primary_concerns`
- `attitude`

These fields should directly influence:

- what gets answered first
- how narrow or broad the claims should be
- whether the tone should reassure, clarify, or de-escalate

# Human Rebuttal Style

Strong rebuttals usually do not read like generic customer-support replies.

Prefer:

- a short global opener
- explicit issue framing
- one strong point per paragraph
- reviewer stance awareness
- a global strategy memo before reviewer-by-reviewer prose

Avoid:

- repetitive gratitude loops
- broad promises
- hiding the answer after too much framing
- sounding equally generic to every reviewer
